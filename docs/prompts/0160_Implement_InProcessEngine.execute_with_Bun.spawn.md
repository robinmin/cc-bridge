## 0160. Implement InProcessEngine.execute() with Bun.spawn

### Background

This is the core implementation task for InProcessEngine. The current implementation is a stub that always returns unavailable. We agreed on Approach 2: Subprocess with Fast Startup - using `Bun.spawn` to launch the gateway binary as a subprocess with a special flag, rather than using Bun worker threads.

References:
- Current stub: `src/gateway/engine/in-process.ts`
- HostIPC implementation reference: `src/gateway/engine/host-ipc.ts`
- LLM provider mechanism: `src/dockers/.env.example`, `scripts/resolve-llm-provider.sh`

### Requirements

1. Implement `isAvailable()` - check if subprocess can be spawned (e.g., check for gateway binary)
2. Implement `execute()` using `Bun.spawn`:
   - Spawn gateway binary with `--in-process` flag
   - Pass prompt via stdin or IPC
   - Collect output from stdout
   - Handle timeout (kill subprocess if exceeds timeout)
   - Handle subprocess crash/exit
3. Return proper `ExecutionResult` with status, output, and error handling
4. Environment variables (LLM_*, ANTHROPIC_*) are inherited automatically from parent process

### Q&A

- Q: How to pass prompt to subprocess? A: Via stdin JSON-RPC
- Q: How to receive output? A: From stdout JSON-RPC response
- Q: What happens on timeout? A: Kill subprocess, return timeout error
- Q: How to handle the gateway entry point? A: Need task 0161 --in-process flag first

### Design

Using subprocess approach (Bun.spawn):
1. InProcessEngine spawns gateway binary with `--in-process` flag
2. Pass ExecutionRequest via stdin as JSON
3. Read ExecutionResult from stdout
4. Handle timeout with subprocess.kill()
5. Environment inherited from parent process

### Solution

Implemented in `src/gateway/engine/in-process.ts`:

1. **isAvailable()**: Returns `true` when enabled (was always returning `false`)
2. **execute()**: Spawns gateway subprocess with `--in-process` flag:
   - Passes request via stdin as JSON
   - Reads response from stdout
   - Handles timeout with subprocess.kill()
   - Inherits all env vars (LLM_*, ANTHROPIC_*) from parent
3. **Gateway path resolution**: Finds gateway binary in dev/prod paths

**Note**: Requires task 0161 (--in-process CLI flag) to be implemented for subprocess to work correctly.
