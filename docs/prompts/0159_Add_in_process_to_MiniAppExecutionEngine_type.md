## 0159. Add in_process to MiniAppExecutionEngine type

### Background

This task is part of enabling InProcessEngine as a mini-app execution engine option. Currently, the mini-app driver validates `execution_engine` and only accepts `claude_container`, `claude_host`, and `codex_host`. We need to add `in_process` as a valid option.

References:
- InProcessEngine stub: `src/gateway/engine/in-process.ts`
- Mini-app driver validation: `src/gateway/apps/driver.ts:145-154`
- MiniAppExecutionEngine type: `src/gateway/apps/driver.ts:25`

### Requirements

1. Add `"in_process"` to the `MiniAppExecutionEngine` union type in `driver.ts:25`
2. Add `"in_process"` to the validation check in `driver.ts:145-154`
3. Add `"in_process"` to the `parseExecutionEngine()` function in `driver.ts:102-107`
4. Add `in_process` handling in `resolveMiniAppSpecPath()` if needed

### Q&A

- Q: Should in_process use a dedicated engine or go through orchestrator? A: Go through orchestrator (will use in-process layer when implemented in task 0160)

### Design

Simple type addition - no architecture changes needed.
