## 0161. Add --in-process CLI flag to gateway

### Background

When InProcessEngine spawns a subprocess, it needs a way to tell the gateway binary to run in "in-process mode" (execute pi-mono and exit) instead of starting the HTTP server. This requires a CLI flag.

References:
- Gateway entry point: `src/gateway/index.ts`
- HostIPC execute pattern: `src/gateway/engine/host-ipc.ts:63+`

### Requirements

1. Add `--in-process` flag to gateway CLI argument parser
2. When `--in-process` is set:
   - Skip HTTP server startup
   - Read prompt from stdin or IPC
   - Execute pi-mono/Claude Code
   - Output result to stdout
   - Exit with appropriate code
3. Design the IPC contract (how to pass prompt, how to receive output)

### Solution

Implemented in `src/gateway/index.ts`:

1. Added `IN_PROCESS_MODE` check at startup (line 32)
2. Added `runInProcessMode()` function that:
   - Reads JSON request from stdin
   - Executes via orchestrator
   - Writes result to stdout
   - Exits with appropriate code
3. When `--in-process` flag is present, skips HTTP server entirely

### Design

- Input: JSON via stdin (`{ requestId, prompt, options }`)
- Output: JSON via stdout (`{ output, error, status, exitCode }`)
- Execution: Via orchestrator.execute()
