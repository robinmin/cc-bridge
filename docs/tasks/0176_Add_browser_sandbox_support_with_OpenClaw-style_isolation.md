## 0176. Add browser sandbox support with OpenClaw-style isolation

### Background

Phase 3 of the sandboxing analysis specifies browser-specific sandbox support inspired by OpenClaw's approach. Browser tools (like magent-browser) have unique isolation needs: they require display access, network connectivity for web requests, but should be isolated from the host filesystem and other processes. This extends the sandbox system with browser-aware container configuration.

### Requirements

1. Browser sandbox profile with appropriate Docker config (display forwarding, network access). 2. Browser-specific resource limits (higher memory for browser processes). 3. Filesystem isolation (no host filesystem access except designated output dir). 4. Integration with existing sandbox executor. 5. Unit tests for browser sandbox profile generation. Success: Browser tools can run in isolated containers with display and network access.
