## 0173. Implement resource limits support for sandboxed tools

### Background

Sandboxed tools need resource constraints to prevent runaway processes from consuming excessive memory, CPU, or spawning too many processes. The analysis document specifies support for memory limits, CPU limits, and PID limits on Docker containers. An initial limits.ts exists at src/packages/agent/tools/sandbox/limits.ts that needs verification and test coverage. Resource limits must translate cleanly to Docker run flags.

### Requirements

1. Memory limit support (e.g., '512m', '1g') with validation of format. 2. CPU limit support (e.g., 0.5, 1.0, 2.0) with reasonable bounds checking. 3. PID limit support to prevent fork bombs. 4. Limits translate to correct Docker CLI flags (--memory, --cpus, --pids-limit). 5. Default limits defined for when user does not specify. 6. Unit tests cover limit parsing, validation, Docker flag generation, and defaults. Success: Limits correctly parsed, validated, and converted to Docker flags.
