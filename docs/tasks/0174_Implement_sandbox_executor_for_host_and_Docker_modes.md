## 0174. Implement sandbox executor for host and Docker modes

### Background

The sandbox executor is the core runtime component that actually runs tool commands either on the host or inside a Docker container. It must use the SandboxConfig, pass through security validation, apply resource limits, and respect per-tool policy decisions. An initial executor.ts exists at src/packages/agent/tools/sandbox/executor.ts. This is the integration point where all Phase 1 components come together into a working execution pipeline.

### Requirements

1. Host mode executor runs commands directly via child_process. 2. Docker mode executor builds and runs docker commands with correct flags. 3. Executor validates config via SecurityValidator before running. 4. Executor applies resource limits from ResourceLimits. 5. Executor respects per-tool policy from ToolSandboxPolicy. 6. Proper error handling for Docker not available, container failures, timeouts. 7. Integration tests cover host execution, Docker execution, validation rejection, and timeout handling. Success: Commands execute correctly in both modes with full security and resource enforcement.
