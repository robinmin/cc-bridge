## 0170. Define SandboxConfig types and Docker configuration interface

### Background

The tool sandboxing system needs a well-defined type system as its foundation. The analysis document at docs/tool-sandboxing-analysis.md specifies a ToolSandboxConfig interface with defaultMode, docker settings (image, network, memoryLimit, cpuLimit, mounts), and toolPolicy map. An initial config.ts exists at src/packages/agent/tools/sandbox/config.ts but needs validation against the analysis spec, proper JSDoc documentation, and export alignment with the rest of the sandbox module.

### Requirements

1. ToolSandboxConfig interface matches the spec from the analysis doc (defaultMode: host|docker, docker settings, toolPolicy map). 2. All Docker-related types properly defined (network: bridge|none, mount definitions). 3. Types exported cleanly from sandbox/index.ts. 4. JSDoc documentation on all public types. 5. Unit tests for type validation helpers (e.g., isValidSandboxConfig). Success: Types compile, tests pass, other sandbox modules can import and use them.
