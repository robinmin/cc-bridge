## 0175. Integrate sandbox module with tool system and add CLI flags

### Background

Phase 1 components need to be wired into the existing cc-bridge tool system. The sandbox module must be imported and used by the tool execution pipeline in src/packages/agent/tools/index.ts. Additionally, the CLI needs --sandbox flag support (strict|permissive|host|docker:<image>) so users can control sandbox behavior. This task bridges the sandbox subsystem with the rest of cc-bridge.

### Requirements

1. src/packages/agent/tools/index.ts imports and uses sandbox executor for tool dispatch. 2. CLI flag --sandbox=strict enables sandboxing for all external tools. 3. CLI flag --sandbox=permissive enables sandboxing only for dangerous tools. 4. CLI flag --sandbox=host disables all sandboxing. 5. CLI flag --sandbox=docker:<image> specifies custom Docker image. 6. Default behavior: no sandbox for built-in tools, sandbox for external. 7. Integration tests verify CLI flags affect tool execution behavior. Success: End-to-end flow works from CLI flag through policy to executor.
