## 0180. memory-storage-foundation

### Background

The memory system needs a solid foundation with the Openclaw-style directory structure. This includes creating the .memory/ root, memory.md for durable facts, memory/YYYY-MM-DD.md for daily logs, and bank/ with typed pages (world, experience, opinions, entities).

### Requirements

Create .memory/ directory structure with memory.md, memory/ subdirectory, and bank/ subdirectory. Implement TypeScript interfaces for MemoryEntry, DailyLog, BankPage. Files must be readable/writable with proper error handling.

### Design


### Design

**Storage Architecture:**
```
.memory/                     # Root memory directory
├── memory.md               # Core durable facts + preferences
├── memory/                  # Daily logs
│   └── YYYY-MM-DD.md       # Daily log entries
└── bank/                    # Typed memory pages
    ├── world.md            # Objective facts
    ├── experience.md       # Agent's experiences
    ├── opinions.md         # Subjective preferences
    └── entities/           # Entity-specific facts
        └── *.md
```

**TypeScript Interfaces:**
```typescript
interface MemoryEntry {
  id: string;
  type: 'fact' | 'preference' | 'context';
  content: string;
  source?: string;
  createdAt: Date;
  updatedAt: Date;
}

interface DailyLog {
  date: string; // YYYY-MM-DD
  entries: LogEntry[];
}

interface BankPage {
  type: 'world' | 'experience' | 'opinions' | 'entity';
  name: string;
  content: string;
  lastUpdated: Date;
}
```

**File Operations:**
- `ensureMemoryDir()` - Create directory structure if missing
- `readMemoryFile(path)` - Read markdown file
- `writeMemoryFile(path, content)` - Write markdown file
- `listDailyLogs()` - List all daily log files
- `listBankPages(type?)` - List bank pages, optionally filtered by type

### Solution

**Approach:** Create the foundation for Openclaw-style memory storage with proper directory structure and TypeScript interfaces.

**Implementation:**
1. Create `.memory/` directory structure at project root
