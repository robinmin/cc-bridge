## 0171. Implement security validation for sandbox configurations

### Background

The sandbox system must block dangerous Docker configurations to prevent security breaches. The analysis document specifies blocking: host network mode, unconfined seccomp profiles, unconfined AppArmor profiles, and requiring absolute paths for bind mounts. An initial validator.ts exists at src/packages/agent/tools/sandbox/validator.ts that needs verification against these security rules and comprehensive test coverage for each blocked configuration pattern.

### Requirements

1. Validator blocks host network mode in Docker config. 2. Validator blocks unconfined seccomp profile. 3. Validator blocks unconfined AppArmor profile. 4. Validator requires absolute paths for all bind mounts. 5. Validator returns descriptive error messages explaining why config was rejected. 6. Unit tests cover every blocked pattern plus valid configurations. Success: All dangerous configs rejected with clear messages, all valid configs pass, 100% test coverage on validation rules.
