## 0177. Implement network isolation options for sandboxed tools

### Background

Phase 3 of the sandboxing analysis specifies network isolation options beyond the basic bridge/none modes. Some tools need full network access, others need access only to specific hosts, and some should be completely network-isolated. This adds granular network policy support to the sandbox system.

### Requirements

1. Network mode 'none' completely isolates container from network. 2. Network mode 'bridge' provides standard Docker networking. 3. Network allowlist support for restricting outbound connections to specific hosts/ports. 4. DNS resolution control within containers. 5. Unit tests for each network mode. Success: Tools can be configured with granular network access policies.
