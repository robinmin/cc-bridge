## 0172. Implement per-tool sandbox policy engine

### Background

Different tools have different security needs. Built-in tools (bash, read-file, write-file, web-search) should run on the host by default, while external/unknown tools should be sandboxed. The analysis document specifies a per-tool policy system where each tool can have custom sandbox requirements (sandbox: boolean, timeout, maxMemory). An initial policy.ts exists at src/packages/agent/tools/sandbox/policy.ts that needs verification against the tool classification rules and integration with the existing tool registry in src/packages/agent/tools/index.ts.

### Requirements

1. Built-in tools (bash, read-file, write-file, web-search) default to host execution. 2. External/unknown tools default to Docker sandbox. 3. Per-tool overrides supported via toolPolicy config map. 4. Policy engine integrates with existing tool registry. 5. Policy lookup is O(1) by tool name. 6. Unit tests cover default policies, overrides, and unknown tool handling. Success: Policy engine correctly classifies all tool types, overrides work, integrates with tool index.
