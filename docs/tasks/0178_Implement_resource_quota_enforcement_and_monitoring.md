## 0178. Implement resource quota enforcement and monitoring

### Background

Phase 3 of the sandboxing analysis specifies resource quota enforcement that goes beyond simple Docker limits. This includes tracking cumulative resource usage across multiple tool invocations, enforcing session-level quotas, and providing usage reporting. This ensures that even within limits, a session cannot consume unbounded resources over time.

### Requirements

1. Session-level resource tracking (total memory-seconds, CPU-seconds used). 2. Configurable session quotas (max total memory, max total CPU time). 3. Quota exceeded handling (block further tool execution, warn user). 4. Usage reporting (current usage vs quota). 5. Unit tests for quota tracking and enforcement. Success: Resource usage tracked across tool invocations, quotas enforced, usage reportable.
